# Proxy Setup

Proxy settings apply to `apps/agent`.

Set:

- `PROXY`
- `PROXY_USERNAME` (optional)
- `PROXY_PASSWORD` (optional, must be set with `PROXY_USERNAME`)

Supported proxy formats:

- `host:port`
- `http://host:port`
- `https://host:port`
- `socks5://host:port`

Credentials must be provided via `PROXY_USERNAME`/`PROXY_PASSWORD` (not inline in `PROXY`).

## Session ID Rotation

If your provider uses sticky proxy sessions in username, the worker auto-injects a unique session id before each browser launch.

Recognized session patterns include:

- `sessid-*`
- `session-*`
- `session_id=*`
- `{{SESSID}}`
- `$SESSID`

On throw/retry, the next launch gets a fresh session id automatically.

## Operational Guidance

- Keep proxy credentials out of source control.
- Track provider-specific failure rates.
- Increase `AGENT_WORKER_CONCURRENCY` only after proxy pool stability is verified.
