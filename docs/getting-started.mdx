# Getting Started

OneGlanse supports three setup paths.

## 1. Use the Cloud App

Use the hosted app if you want the fastest setup:

- App: `https://app.oneglanse.com`
- Landing: `https://oneglanse.com`
- Docs: `https://docs.oneglanse.com/`

Recommended flow:

1. Sign in
2. Create or select a workspace
3. Add prompts
4. Review responses, sources, and dashboard metrics

## 2. Run It Locally

Use local mode when you want to run the web app and agent on your own machine.

### Requirements

- Node.js 20+
- pnpm 10+
- Docker + Docker Compose

### Install & Start

```bash
pnpm install
pnpm local
```

This:

- Creates `.env` if missing (with auto-generated secrets)
- Bootstraps the Camoufox browser runtime automatically on first run
- Starts Postgres, ClickHouse, and Redis
- Runs database migrations
- Starts the web app and the agent
- Opens the app at `http://localhost:3000`

### Local Notes

- Local mode supports interactive provider auth via `/providers`
- Local mode supports manual prompt runs
- Local mode does not support recurring scheduling

## 3. Self-Host It

Use self-host mode when you want the full product in your own infrastructure.

### Install & Start

```bash
pnpm install
pnpm self-host
```

This starts:

- The main app stack (`docker-compose.yml`) — web, agent, Postgres, ClickHouse, Redis
- The always-on public stack (`docker-compose.public.yml`) — Nginx gateway, landing page
- Tries to pull the latest published images before starting
- Falls back automatically to a local Docker build if pulling is unavailable

Default ports:

- Landing: `3000`
- App (via gateway): `3001`
- Auth upload API: `3333`

Persistent state defaults to `/opt/oneglanse/storage`. Override with `ONEGLANSE_STORAGE_ROOT` if your VPS should store data elsewhere.

### Docker Image Overrides

To use custom Docker images instead of the published ones, set these in `.env` before running `pnpm self-host`:

- `ONEGLANSE_WEB_IMAGE`
- `ONEGLANSE_AGENT_IMAGE`
- `ONEGLANSE_POSTGRES_IMAGE`
- `ONEGLANSE_LANDING_IMAGE`

Use `pnpm self-host:build` only if you intentionally want to build images from local source.

### Provider Auth Setup

Interactive provider login cannot run on a headless server. Capture sessions locally and transfer them to the VPS.

**Step 1 — Start your VPS first**

```bash
pnpm self-host
```

The auth upload API must be running on port `3333` before you transfer sessions.

**Step 2 — Transfer sessions from your local machine**

If `ONEGLANSE_VPS_IP` and `AGENT_AUTH_UPLOAD_TOKEN` are set in your local `.env`, run:

```bash
pnpm upload:vps
```

This uploads any existing local sessions directly without reopening the auth flow.

If you don't have local sessions yet, or want to capture new ones:

```bash
pnpm auth -- --upload-url http://YOUR_VPS_HOST:3333/auth/sessions --upload-token YOUR_TOKEN
```

This opens the local `/providers` page, lets you connect providers interactively, and uploads the saved sessions to the VPS.

**Step 3 — Verify transfer**

The agent reads transferred sessions at the start of each job run and reseeds the browser profile automatically if the session has changed.

### Self-Host Notes

- Re-run the transfer command any time a provider session expires
- Recurring scheduling is supported in self-host mode
- App-only updates: `pnpm self-host:app`
- Public stack updates: `pnpm self-host:public`
